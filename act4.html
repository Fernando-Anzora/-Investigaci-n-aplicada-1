Buenas practicas de JavaScript:

1.Nombrar adecuadamente las partes del código. Organización y legibilidad del código representan los mejores resultados a la hora de realizar esta práctica, pues el nombrar, clasificar o definir de forma elocuente, variables, funciones, constantes, los objetos y demás, nos ayudan a tener una visión clara de nuestro código, mas aun y cuando se trata de un código extenso, donde el nombrar alguna variable de forma monótona con una letra o un número, será un caos, pues se tendrá de revisar el código parte por parte hasta en encontrar el significado de esa letra dentro de nuestro código. Nombrar con palabras claves o representativas nos ayudaran de guía en nuestro código Js.

2.Emplear la triple igualdad para realizar comparaciones. Js cuenta con métodos para poder realizar la comparación de manera sencilla, no obstante partes de estos métodos como por ejemplo el de la doble de negación, en el que se complica de manera significativa parte del proceso que se pudiera resolver de manera sencilla con la triple igualdad, veamos pues el siguiente ejemplo, donde se emplea la doble igualdad para poder ingresar el precio de algo, pero al tratar de ingresar el número 0, seguirá mostrando el mensaje en consola “introduce el precio” a pesar de haberlo ingresado.
