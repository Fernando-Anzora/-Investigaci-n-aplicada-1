¿Qué es Linting?

Bueno hay que empezar primero por ver el que la mayoría de programadores no son magos o eruditos y pueden llegar a tener algún tipo de error en el código y bueno sabemos que estos errores hay diferentes tipos como los que dañan la interfaz otros que no aseguran que haya una seguridad del sistema y bueno la más común es que un simple error hace que la app deje de funcionar y bueno de forma mas resumida estos son algunos errores

1.errores de sintaxis
2.Código poco intuitivo o dificil de mantener
3.Uso de "malas prácticas"
4.uso de estlios de codigo inconsistentes.


Y bueno este término ha día de hoy se utiliza para algunas herramientas que puedan realizar cualquier tipo de corrección de algún tipo de error y bueno la función de estas herramientas está sencillo como que analiza un análisis al código incluso hasta pueden darte algún tipo de sugerencia de corrección de errores


Sintaxis:

Bueno ahora daremos un ejemplo de este tipo de herramientas y que es mayormente utilizaba en javascrips ESLint es una herramienta de código abierto enfocada en el proceso de "lintig" para javascript (o más correctamente para ECMAScript). ESLint es la herramienta predominante para la tarea de "limpiar" código javascript tanto en el servidor (node.js) como en el navegador.

Dado que javascript es un lenguaje dinámico y de tipado débil, es especialmente fácil caer en errores humanos a la hora de escribir código. Y bueno en resumidas cuentas mostraremos una lista de los errores que resuelve esta herramienta.

1.Mostrarte errores de sintaxis.
2.Mostrarte errores cuando no se siguen buenas prácticas.
3.Proveer sugerencias para mejorar tu código.
4.Mantener un estilo consistente en tu código o reforzar reglas internas de tu propio equipo.
